Qu'est-ce que hawser

haw est une plateforme multi-dépôts pour les équipes qui livrent un produit réparti sur de nombreux dépôts Git. Cinq capacités, un seul binaire — chacune résolvant une facette du problème multi-dépôts.

🧱

Composer

Une stack reproductible à partir de plusieurs dépôts : un manifeste haw.toml et un haw.lock commité épinglent chaque dépôt sur un SHA exact. Stacks & overlays, clone superficiel et partiel.

⚙️

Orchestrer

Exécute le travail sur toute la flotte : run/build/test en parallèle, un grep à l'échelle de la flotte, et une passerelle verify qui sort en 3 en cas de dérive — à brancher directement dans la CI.

🔀

Collaborer

Un flux de changements inter-dépôts, inter-forges : une fonctionnalité = une branche sur N dépôts, avec des PR/MR liées sur GitHub, GitLab & Bitbucket, et land pour merger dans l'ordre des dépendances.

🚁

Piloter

Un cockpit TUI façon k9s (simplement haw) : lire → creuser → agir. Grille de flotte en direct, checks de PR, avancement CI — puis merger, approuver ou checkout au clavier.

🛡️

Gouverner

Chaîne d'approvisionnement & audit, intégrés : plugins, SBOM (CycloneDX + SPDX), provenance SLSA/in-toto, signature cosign/minisign, et bundles evidence pour la qualification.

Ce n'est pas un wrapper git. C'est la couche au-dessus de git : le manifeste, le lockfile, le build/test/CI à l'échelle de la flotte, le flux de changements inter-forges, et la piste d'audit.

Démarrage rapide

Trois commandes, de l'installation au cockpit.

# installe le binaire unique
$ cargo install hawser
# clone chaque dépôt, écrit haw.lock
$ haw sync
# ouvre le cockpit
$ haw

Nouveau ici ? Lance le cours guidé ci-dessous, ou explore des manifestes exécutables dans examples/.

Pourquoi hawser · pour qui

Répartir un produit sur plusieurs dépôts est courant — et indépendant du domaine. Le manifeste, le lockfile, le flux de changesets, le build/test de la flotte et les hooks de gouvernance sont les mêmes dans chaque domaine ; seuls les dépôts et les commandes changent.

Embarqué & automobile

HAL/BSP/MCAL partagés et réutilisés sur de nombreux ECU ; dépôts de config AUTOSAR/ARXML épinglés à côté du code.

Microservices backend

Une fonctionnalité couvrant N services plus un proto/lib partagé — branchés, en PR, et landés ensemble.

Plateformes ML / data

Dépôt de modèle + dépôt de pipeline de données + infra de serving, épinglés comme une seule baseline reproductible.

Plateforme / infra

Dépôts Terraform, Helm et modules réutilisables composés et versionnés comme un tout.

Mobile

Un dépôt d'app et son ou ses dépôts SDK, modifiés de concert dans un seul changeset.

« Ce n'est pas un wrapper git. haw orchestre Git et les API des forges — il ne réimplémente pas le moteur de merge de Git, ne remplace pas une forge, ni ta toolchain. »

Installer

Choisis un gestionnaire de paquets — tous installent le même binaire haw v0.1.3.

# Rust / crates.io (canonique)
$ cargo install hawser
# macOS + Linux (Homebrew)
$ brew install nastwinns/tap/hawser
# Windows (Scoop)
$ scoop bucket add nastwinns https://github.com/Nastwinns/scoop-bucket
$ scoop install hawser
# Entièrement statique — pas de glibc, pas de runtime ; idéal serveurs, conteneurs, air-gap
$ curl -sSL https://github.com/Nastwinns/hawser/releases/download/v0.1.3/\
haw-0.1.3-x86_64-unknown-linux-musl.tar.gz | tar xz \
  && sudo install haw /usr/local/bin/

Chaque release est signée (cosign .sig/.pem + .sha256). Matrice complète & vérification : docs/INSTALL.
